Set

发表于 2019-06-15 | 分类于 java

   List是java集合中有序的代表，Set则是无序集合的代表，面试中也会经常拿这两者来进行比较。今天就来看下Set中的HashSet和TreeSet。

HashSet

   HashSet是Set集合中使用最多的对象，它用来保存一组不可重复的相同类型的数据。

实例化

   我们先来看下HashSet的构造函数:

private transient HashMap<E,Object>map;

可以看到，在HashSet的构造函数中实例化了一个HashMap对象，这说明在HashSet内部是通过HashMap来存储数据的。

add

   知道了内部使用的数据结构，很多东西就可以直接推测出来了，add肯定是调用map的put方法来实现的：

在HashSet中，将add的值作为内部HashMap的key来进行存储，在map中，每个key经过hash都有一个确定的位置，通过这种方式，保证了set内部数据的不重复。

TreeSet

   TreeSet也是用来保存一组不可重复的相同类型的数据，但是内部保证了数据的有序性。

实例化

   HashSet底层使用HashMap是来实现，可以猜测TreeSet内部是不是也使用了某种Map来存储数据呢，想要数据有序，就只有TreeMap是满足要求的了：

观察源码证实了上面的推测。

add

   添加数据则同样利用了TreeMap提供的put方法，我们知道TreeMap之所以可以保证有序，是通过外部排序器Comparator来实现的，java中的数据结构大部分都实现了这个接口，所以可以直接用，如果想要自定义一个对象使用TreeSet或者TreeMap进行存储的话，一定要保证他们已经实现Comparator接口，并重写了compare()方法。

总结

   HashSet和TreeSet都是内部元素不能重复的集合，HashSet底层利用HashMap来实现，TreeSet底层利用TreeMap来实现有序，前提是对象实现了外部排序器Comparator.
