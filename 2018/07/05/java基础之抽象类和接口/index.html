java基础之抽象类和接口

发表于 2018-07-05 | 分类于 java

   抽象类和接口的使用在java开发中是比较常见的，今天我们就来比较下接口类和抽象类的之前的联系和区别。

抽象类

什么是抽象类？

   使用abstract修饰符来修饰的类就是抽象类，使用abstract修饰的方法就叫抽象方法，抽象方法没有方法体。抽象类除了不能实例化以外，其他都和普通的java类一样。

抽象类需要遵守的规则

  1. 抽象类必须使用abstract修饰符来修饰；
  2. 抽象类不可以被实例化，不能通过new来调用构造器创造实例；
  3. 抽象类中只能有成员变量，构造器，方法（普通方法和抽象方法都可以），初始化块，内部类（接口和枚举），构造器主要用来被子类调用；
  4. 以下三种情况只能被定义成抽象类：
  • 内部有抽象方法；
  • 继承了抽象类但没有完全实现父类的抽象方法；
  • 实现了接口但没有完全重写接口中的方法；

使用抽象类的好处

   抽象类可以更好的发挥java的多态性，建立良好的继承层次。

接口：

什么是接口？

使用interface关键字代替class的‘类’就是接口，接口是一个极度抽象的类，这个这个类里面只定义一系列规范，没有任何的具体实现。

接口需要遵守的规则

接口中只能有成员变量（静态常量），方法（抽象方法，类方法和默认方法），内部类（接口和枚举）
*类方法和默认方法，jdk8才支持
接口中的方法默认的修饰符都是public abstract
接口中的常量默认修饰符是public static final

使用接口的好处

  • 面向接口编程，实现松耦合；
  • 可以实现多个接口达到组合的目的；
  • 接口可以多继承；

抽象类和接口的区别：

相同处：

  1. 都不能被实例化；
  2. 里面的抽象方法都需要被继承或者实现的子类重写。

区别：
抽象类和接口的区别主要在设计目的上：
抽象类体现的是模板设计的理念，多个子类抽取出一个父类，实现了一部分方法，个性化的方法交由子类自己实现。
接口则是定义一系列规范给各组件，体现的是模块之间的解耦。
