java多线程（五）线程通信

发表于 2018-07-09 | 分类于 java

    java中提供了三种方式进行线程通信：

使用同步监视器控制线程通信

    我们在线程同步中已经介绍过同步监视器主要就是通过同步方法和同步代码块实现的，同步监视器中的线程通信就是通过调用同步监视器的wait(),notify()和notifyAll()方法实现。
    线程通信中有个经典的生产者消费者模式，我们用同步监视器的方式是来实现生产者消费者模式，看一下是怎么工作的：

   一般都会使用notifyAll()而不是notify()，因为notify()有可能会导致死锁。

使用Condition对象控制线程通信

   我们知道线程同步除了使用同步监视器以后，还可以通过lock对象显式的加锁，释放锁。这样的同步也就无法使用同步监视器对象的wait(),notify()和notifyAll()方法，所以java提供了condition对象来控制线程通信，confition依赖与lock对象，提供了await(),signal()和signalAll()方法，还可以为多个线程建立不同的condition对象。

  • ArrayBlockingQueue:基础数组实现
  • LinkedBlockingQueue:基于链表实现。
