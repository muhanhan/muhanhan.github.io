java基础之注解

发表于 2018-08-07 | 分类于 java

    在我们日常的开发过程中，会经常遇到注解，比如重写方法时的@Override,使用springmvc中用到的@Controller,@Autherwide,@RequestMapping等等，注解在代码中就好像一个标签，通过这个标签我们或者编译器可以获取制定的信息，也可以通过这个注解来实现特定的功能。

认识注解

元注解

如果点进上面的几个注解查看源代码，可以发现在这个注解中还使用到了其他注解，这些用来声明注解时使用的注解被称作元注解。

java目前共有四个元注解：

@Target

用于指定该注解使用的位置，是在包上，还是方法上

@Retention

用于指定注解生效的作用域，源码/编译／运行时

@Inherited

该注解可被子类继承

@Documented

可以被javadoc工具编译成api

Retention的枚举值：

  • RetentionPolicy.SOURCE:该注解保留在源码阶段，编译时就会被丢弃，不会加载到jvm中；
  • RetentionPolicy.CLASS：该注解保留到编译时，但是不会加载到jvm中；
  • RetentionPolicy.RUNTIME：该注解保留到运行状态，加载至jvm中；

Target的枚举值：

ElementType.TYPE:可以作用在类型上，如类，接口，枚举

  • ElementType.FIELD：可以作用在成员变量上
  • ElementType.METHOD：可以作用在方法上
  • ElementType.PARAMETER：可以作用在参数上
  • ElementType.CONSTRUCTOR：可以作用在构造器上
  • ElementType.LOCAL_VARIABLE：可以作用在局部变量上
  • ElementType.ANNOTATION_TYPE：可以作用在注解上
  • ElementType.PACKAGE：可以作用在包上
